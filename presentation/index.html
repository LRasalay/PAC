Policy Schema

Flexible JSON-based policy definitions with powerful rule expressions

  • check_circle JSONPath-based field access
  • check_circle Boolean logic (all, any, not)
  • check_circle Conditional rules (when/then)
  • check_circle Date and temporal operators
policy.json 
{
  "id": "mrm-model-doc",
  "version": "1.0.0",
  "metadata": {
    "name": "Model Documentation",
    "domain": "MRM",
    "severity": "high"
  },
  "rules": [{
    "id": "doc-001",
    "conditions": {
      "path": "$.model.documentation",
      "operator": "exists"
    }
  }]
}

REST API

Simple, powerful API for policy management and evaluation

  • description Policy CRUD operations
  • play_circle Single & bulk evaluation
  • history Evaluation history & audit

evaluate.sh 
# Evaluate data against a policy
curl -X POST /api/v1/evaluate \
  -H "Content-Type: application/json" \
  -d '{
    "policy_id": "mrm-model-doc",
    "data": {
      "model": {
        "id": "credit-v2",
        "documentation": {...}
      }
    }
  }'

# Response
{
  "compliant": true,
  "results": [...]
